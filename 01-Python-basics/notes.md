## 🧠 What I Learned

- **VS Code setup:** How to set up and use VS Code as my Python coding environment (installing extensions, running scripts, etc.)
- **Syntax & semantics:** The basic rules for how Python code is written and structured — indentation, comments, print statements.
- **Variables:** How to create and name variables to store data, and Python's rules around naming them.
- **Data types:** The core built-in types — e.g. `int`, `float`, `str`, `bool` — and how Python treats each differently.
- **Operators:** Arithmetic (`+ - * /`), comparison (`== != > <`), and logical (`and or not`) operators, and how to use them in expressions.

## 💡 Key Takeaways

- Getting VS Code set up felt intimidating at first, but once I saw how simple it is to write and run a script, it clicked fast. The biggest realization was how much Python cares about clean, readable structure — indentation isn't just style, it's actually part of the syntax. Variables felt intuitive since it's just "naming a box to store something," but understanding the different data types (int, float, str, bool) made me realize Python is quietly tracking what kind of data I'm working with behind the scenes. Operators were the fun part — seeing how == (comparison) is different from = (assignment) was a small but important "aha" moment. Overall, this module built the foundation — nothing complex yet, but I can already see how these basics (variables + data types + operators) are the building blocks for everything else coming up.

## 🚧 Challenges

Getting VS Code set up and configured correctly took longer than expected — figuring out the right extensions and making sure the Python interpreter was properly selected was a bit confusing at first.

Keeping track of indentation was tricky in the beginning — a misplaced space caused errors before I understood that Python actually depends on it structurally, unlike some other languages.
Remembering the difference between = (assignment) and == (comparison) took a couple of mistakes before it stuck.

Knowing when to use each data type wasn't always obvious right away — I had to slow down and think about what kind of value I was actually working with before choosing the right one.
